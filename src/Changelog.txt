=== Versão 2.1 ===
Ajustes para o CMS unificado funcionar na remaza

Funcionalidades:
- GetUserProfile no IContext
- SiteBaseController não foi incluso na versão anterior
- Método RequestCookieDictionary no IContext
- Método Md5HashHexa que gera output em hexastring ao invés de guid

Fixes:
Interface IUserStore estava duplicada
GetDescripton do extensor de enumeradores agora tem restrição de tipo (para facilitar inferência)

Breaking Changes:
- WebModule precisa ficar em um assembly separado. Ele é usado para extensões do admin do CMS e para evitar conflitos
 de assemblies ou dificuldades de inferência, precisa ser a mesma dll utilizada no Admin e na extensão. Manter um
 conjunto menor de items que mudam com pouca freqüência é o único caminho viável conhecido.

==== Versão 2.0 ==== 
Incorpora alterações feitas para a Remaza

Funcionalidades:
- Implementação de cache em memória.
- Incluindo Dapper (ORM)
- Conversor de DateTime para Decimal (ddmmyyyy)
- KeyValue (para conveniencia ao invés do keypairvalue que é passado por valor)
- Método .ToBytes() para strings
- Validador de CPF e CNPJ
- Atributos para validar CPF e CNPJ
- Adicionado Wrappers ( Transparent, Cache, WCF )
- Atributo "RequiredIf"
- AdminBaseController movido para o Practices
- Criação do SiteBaseController
- SideMenuActionForItemViewModel movido para o Practices

Breaking Changes:
- Cache do couchbase movido para outro pacote e renomeado para CouchBaseActiveCache
- IContext tem a assinatura do método SetAuthenticated alterada.
- WebContext foi movido para o namespace MTO.Practices.Common.Web.
- TestContext foi movido para o namespace MTO.Practices.Common.Debug.
- Practices.Config removido da solução.
- EventException movido do Practices.Logger.Elmah para o Practices.Common
- ILogger reformulado.
- NullLogger movido para MTO.Practices.Common.Logging

BugFixes:
- Possível loop infinito entre listeners aninhados
